# ready_data/ README

This directory stores **exported GRAVE-D datasets** that are intended
for use by other repositories (e.g., `data-2025`,
                               `autocracy_conflict_signaling`). Files here are regenerated by
`R/05_build_master.R` and should not be edited manually.

## Core GRAVE-D outputs

| File name                          | Unit of analysis | Description                                                  | Produced by script     | Consuming repos / uses                                           |
        |------------------------------------|------------------|--------------------------------------------------------------|------------------------|------------------------------------------------------------------|
        | GRAVE_D_Master.csv                 | dyad-year        | Full GRAVE-D dyadic dataset with core variables and controls | 05_build_master.R      | `data-2025` (base dyadic dataset), `autocracy_conflict_signaling` (main analysis data) |
        | GRAVE_D_Master_with_Leaders.csv*   | dyad-year        | GRAVE-D dyadic dataset with expanded leader-level attributes | 05_build_master.R (or follow-on export script) | `data-2025` and `autocracy_conflict_signaling` where leader variables are required |
        
        \* If you prefer to keep a single export, you can either:
        - Export only `GRAVE_D_Master.csv` and merge leader variables in the
consuming repo, or
- Make `GRAVE_D_Master_with_Leaders.csv` the canonical exported file
and point all consuming code to it.

## Usage notes

- `grave_d_data2026` is the **producer** repository. Other projects
should treat files in `ready_data/` as inputs, not modify them.
- When updating GRAVE-D:
        1. Pull latest changes in `grave_d_data2026`.
2. Re-run `R/00–05_*.R` to regenerate `GRAVE_D_Master.rds` and the
`.csv` exports in `ready_data/`.
3. Commit the updated `.csv` files if they are part of your archival
or replication bundle.
- In consuming repos:
        - `data-2025`: load `ready_data/GRAVE_D_Master_with_Leaders.csv`
(or `GRAVE_D_Master.csv` if you merge leaders there) as the base
dyadic dataset.
- `autocracy_conflict_signaling`: use the same exported file as the
main analysis dataset for models and Quarto documents.
